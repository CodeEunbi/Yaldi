### 요구사항:
1. 각 테이블마다 정확히 {row_count}개의 행을 생성해주세요.
2. INSERT 문만 생성하고, 다른 설명이나 주석은 포함하지 마세요.
3. 제약조건을 반드시 준수해주세요:
   - PRIMARY KEY + AUTO_INCREMENT: INSERT 문에서 해당 컬럼 생략
   - PRIMARY KEY (AUTO_INCREMENT 없음): 1부터 시작하는 순차 값
   - UNIQUE: 중복되지 않는 값 생성
   - NOT NULL: NULL이 아닌 값 생성
   - FOREIGN KEY: 참조 테이블의 기존 값 사용

4. 데이터 타입에 맞는 현실적인 값 생성:
   - VARCHAR/TEXT: 의미 있는 문자열 (예: 이름, 이메일, 설명)
   - INT/BIGINT: 적절한 범위의 숫자
   - DATE/DATETIME/TIMESTAMP: 현실적인 날짜/시간
   - BOOLEAN: true 또는 false
   - ENUM: **[매우 중요]** 컬럼에 [허용값: ...] 형식으로 표시된 값만 사용해야 합니다.
     예: provider 컬럼의 [허용값: 'GOOGLE', 'KAKAO']이면 반드시 'GOOGLE' 또는 'KAKAO'만 사용

5. 컬럼명을 고려한 현실적인 데이터:
   - email: 유효한 이메일 형식 (예: user1@example.com)
   - name, username: 실제 이름처럼
   - age: 1-100 사이 값
   - price, amount: 양수 값
   - description: 의미 있는 설명 문구
   - provider, status, type 등: [허용값]에 명시된 값만 사용 (절대 임의로 만들지 말 것!)

6. 외래키 관계가 있다면 참조 무결성을 유지해주세요.

7. 출력 형식:
   - 각 INSERT 문은 한 줄로 작성
   - 세미콜론(;)으로 문장 종료
   - 문자열 값은 작은따옴표(')로 감싸기

### 예시:
INSERT INTO users (email, name, age, provider) VALUES ('john.doe@example.com', 'John Doe', 28, 'GOOGLE');
INSERT INTO users (email, name, age, provider) VALUES ('jane.smith@example.com', 'Jane Smith', 34, 'KAKAO');
